Config Git

Oct 14, 2017 in common

Git (/ɡɪt/) is a version control system for tracking changes in computer files and coordinating work on those files among multiple people. It is primarily used for source code management in software development

—wikipedia


1、设置Git的user name和email：(如果是第一次的话)

2、生成密钥

连续3个回车。如果不需要密码的话。
最后得到了两个文件：id_rsa和id_rsa.pub,如果不是第一次，就选择overwrite：yes

3、添加密钥到ssh-agent

确保ssh-agent是可用的。ssh-agent是一种控制用来保存公钥身份验证所使用的私钥的程序，其实ssh-agent就是一个密钥管理器，运行ssh-agent以后，使用ssh-add将私钥交给ssh-agent保管，其他程序需要身份验证的时候可以将验证申请交给ssh-agent来完成整个认证过程。

添加生成的 SSH key 到 ssh-agent。

4、登陆Github, 添加 ssh 。

把id_rsa.pub文件里的内容复制到github setting的ssh设置中

5、测试：

你将会看到：

选择 yes

如果看到Hi+你的用户名，就说明成功了。

6、发布

note: git 与 svn最大的区别在于，git会保存每个版本的文件快照，便于恢复，svn以及其他版本控制器会保存文件的改动信息。

前两步只在第一次推送时执行，已经建立远程库的不需要重复执行
如果需要更换远程仓库地址，先执行git remote rm origin然后重新第二行代码即可
