jenkins

Apr 05, 2018 in Java

Auto-build tool from git to k8s

简介

与Git无缝贴合，通过githook获取项目更新通知，完成编译-发布动作。

安装

环境：centos

首先需要给yum引入源。

之后安装两个命令：

这里使用git作为更新管道，所以需要安装配置全局git账户git，安装java。测试git，clone一下。再安装maven
允许开机启动

开启端口，访问服务器8080端口.

配置

首先进入界面，回要求输入密码，密码在服务器上，网页上标注了地址，cat一下就好。

然后就可以创建项目了，选择一个自由的项目就可以，我们用git项目，所以勾选GitHub project，并把项目地址录入。

Source Code Management选择git，url还是刚的项目地址，Credentials这里的类型选择用户名和密码，输入的就是你github的账号密码。

Build使用xecute shell，执行maven构建命令，比如mvn clean package

Build Triggers选择GitHub hook trigger for GITScm polling。githook的配置稍后再说。

Build Environment里选择一个Add timestamps to the Console Output，可以看到命令行输出。

Post-build Actions需要插件deploy to container这个插件，这个可以稍后配置，安装插件要重启。

这样我们的项目就配置好了

关于git的配置，先要弄一个Personal Access Token,在Settings/Developer settings/Personal access tokens里，生成一个新的token，对应项目的地址。

到项目的setting里配置webhook。url就是jenkins的服务器地址：8080+/github-webhook/ 结尾要有/，否则可能出问题

回到jenkins，到系统设置的GitHub /Github Servers配置服务，配置用户时使用Secret text权限。测试一下连接。

以上就配置好了当git接受新push的时候，自动构建，测试项目。

下一步就是发布
