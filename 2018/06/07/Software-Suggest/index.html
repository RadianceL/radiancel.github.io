Software Suggest

Jun 07, 2018 in Java

When I design a interface platform system.
I find my system is not elegant realization.
Detrimental to maintenance.
So this my summary.

Three Layer Architecture

Data access layer + Business logic layer + Presentation layer

When I design interface platform system, I made a DTO to result.
But it go through business logic layer straight up controller.

and all data assemble at business logic layer,When I finish this system.
Oh, shit!
A lot kind of log, exception, DAO at here.

Although all service implements method result the same type.
but it also not good for read.

so i reflect,

  1. data assemble should in another class with static method. this method will cache all possible throws exception.
  2. data parsing should in another class with static method. also ache all possible throws exception.
  3. the system should has itself exception, like FooException as root Exception, all other exception extends the root exception.
  4. the system should has itself error code. and some solution for those code.
  5. business logic layer should clear. just communicate with database.
  6. all controller result should have a agreement. structural consistency.

About Design pattern

Just make a good way to coding, if unnecessary, do not use Design pattern
if you coding experience is not enough, at last 5 years maybe.
It well let you can’t coding.
